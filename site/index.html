MLX Porter Runbook

Port PyTorch projects to MLX without guessing.

A practical runbook for agents converting CUDA/PyTorch and Hugging Face models to Apple MLX: inspect first, prove parity, then optimize with receipts.

Use the MLX model porting skill.

1. Inspect this model statically.
2. Choose the closest runbook.
3. Build a source oracle.
4. Port the smallest eager MLX path.
5. Validate weights and parity.
6. Only then optimize.

Why this exists

Large frontier models can often discover useful conversion and optimization steps at runtime. This repo distills that work into a repeatable path, so smaller open-source and local models can follow the same discipline one step at a time.

It is not a prompt dump

The compact skill delegates detail into architecture runbooks, structured assets, and scripts an agent can inspect and execute.

It resists folklore

CUDA techniques are research candidates until there is a demonstrated MLX path, validation gate, and rollback condition.

It keeps claims honest

Performance claims need hardware, versions, workload, baseline, and a quality gate. A pretty speedup without parity is not a result.

The runbook agents follow

The path is intentionally boring. That is the point. Every step narrows uncertainty before the next step spends complexity.

1 Inspect and classify Read config, headers, source files, license, remote-code risk, and architecture signals without executing untrusted model code.
2 Select a runbook Route to dense decoder, ASR, diffusion, audio codec, VLM, SSM, MoE, CV, graph, or another proven family path.
3 Build the source oracle Freeze deterministic fixtures and capture intermediate tensors at meaningful semantic boundaries.
4 Port the minimal graph Implement the smallest eager MLX path first. No quantization, compile, batching, or custom kernels yet.
5 Validate, profile, optimize Account for weights, pass parity, profile bottlenecks, and change one optimization dimension at a time.

Common entry points

People rarely arrive with a clean model-porting brief. These patterns turn familiar PyTorch surfaces into an MLX plan.

Transformer attention QKV maps, RoPE, masks, KV cache, fast SDPA after parity.
Checkpoint loading Safetensors/static intake, deterministic key transforms, no silent missing keys.
Whisper-style ASR Waveform, log-mel, encoder states, token IDs, WER/CER, timestamps.
Diffusion blocks One denoiser timestep, scheduler update, fixed-seed latent trajectory.
LoRA fine-tuning Target modules, trainable membership, scalar loss, gradients, adapter resume.
Quantization Quality gates first, then weight/KV/mixed-bit recipes with rollback.
Serving Prompt cache, batching, speculative decoding, streaming, and API behavior.
Packaging Provenance, license, tokenizer, recipe, smoke test, benchmark metadata.

Rules that keep the port real

Source first

Pin repository, revision, artifacts, tokenizer or processor, license, and checksums before writing conversion code.

Parity before speed

Tensor checks, intermediates, end-to-end outputs, task quality, cache/state, and boundary cases come before optimization.

Explicit weights

Every rename, transpose, reshape, split, concat, cast, quantization state, and tied weight belongs in a testable map.

Measured optimization

Use MLX-native operations, cache design, layout changes, and compile before custom kernels. Keep rollback conditions close.

Start from the offline pipeline

The repo ships tiny synthetic fixtures so an agent can exercise routing, planning, recommendation, and validation without network or Apple Silicon.

python3 mlx-model-porting/scripts/inspect_model.py tests/fixtures/models/decoder --output /tmp/inspection.json
python3 mlx-model-porting/scripts/make_port_plan.py /tmp/inspection.json --output /tmp/PORT_PLAN.md
python3 mlx-model-porting/scripts/recommend_optimizations.py /tmp/inspection.json --markdown /tmp/OPTIMIZATIONS.md
python3 mlx-model-porting/scripts/audit_skill.py --strict mlx-model-porting
python3 -m unittest discover -s tests -v

MLX Porter is a runbook and validation spine for Apple MLX model ports. It does not claim a real model is ported until source provenance, parity, quality, benchmark metadata, and rollback conditions exist.
