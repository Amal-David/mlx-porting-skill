A local MLX optimization receipt must identify the Mac model, chip, unified memory, macOS version, Python version, MLX version, MLX-LM version, model id, model revision, quantization state, prompt token count, generated token count, sampling parameters, cache state, warmup count, measured run count, median prompt tokens per second, median decode tokens per second, peak MLX memory, baseline label, speedup ratio, workload caveat, quality gate, rollback condition, and whether the claim is measured together or a multiplicative hypothesis. The reviewer must reject any recommendation that promotes a source-reported claim to local_reproduced without a reproducible command, raw receipt, and validation gate.

The dense decoder stack contains native low-bit weight quantization, fast scaled-dot-product attention, stable-region compile, prompt-prefix cache, uniform KV quantization, and draft-model speculation. The stack can only become measured_together after one end-to-end receipt enables the relevant runtime knobs at the same time. Separate receipts are useful, but they do not prove composition.
