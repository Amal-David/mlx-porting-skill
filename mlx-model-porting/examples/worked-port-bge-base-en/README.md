# Worked port: BAAI/bge-base-en

This is a real offline F32 port of the cached `BAAI/bge-base-en` BERT encoder
snapshot `b737bf5dcc6ee8bdc530531266b4804a5d77b5d8` to the generated standalone
MLX encoder package on an Apple M4 Pro. The repository retains only small
manifests, the complete weight map, and the parity report; source/converted
weights and NPZ tensor captures stay under `$WORK`.

## Result

- Static inspection routed 200 tensors to `encoder-transformer` and validated
  the exact BERT topology: 12 bidirectional layers, absolute learned positions,
  token types, embedding LayerNorm, and a dense+tanh CLS pooler.
- Conversion mapped 199 parameter tensors one-to-one in F32. The sole ignored
  source tensor is `embeddings.position_ids`, a deterministic non-parameter
  buffer regenerated by the MLX graph.
- The masked fixture `[101, 7592, 2088, 999]` with attention mask `[1, 1, 1, 0]`
  passed all 17 rungs: exact IDs and mask, embedding output, all 12 layer
  outputs, final hidden state, and pooled output.
- The observed maximum absolute drift was `3.544241189956665e-5` at layer 10;
  the recorded F32 policy is `atol=5e-5`, `rtol=1e-4`.

## Offline materialization and provenance

The Hugging Face snapshot uses symlinks into the local blob store, so it is
materialized before inspection. The upstream model card declares MIT terms but
places that declaration beyond the intake reader's bounded frontmatter window;
the materialized config therefore records the same operator-verified
`license: mit` declaration. This changes only the local intake sidecar, not the
checkpoint tensors.

```bash
export WORK="$HOME/.cache/mlx-porting-work/bge-base-en-run"
export SNAPSHOT="$HOME/.cache/huggingface/hub/models--BAAI--bge-base-en/snapshots/b737bf5dcc6ee8bdc530531266b4804a5d77b5d8"
export MODEL="$WORK/source"
export HF_HUB_OFFLINE=1
export TRANSFORMERS_OFFLINE=1

mkdir -p "$MODEL"
cp -LR "$SNAPSHOT"/. "$MODEL"/
# Add `"license": "mit"` to the materialized config, matching README.md.
```

No command uses a Hub ID, enables remote code, or accesses the network.

## Inspect and scaffold

```bash
python3 mlx-model-porting/scripts/inspect_model.py "$MODEL" \
  --output "$WORK/inspection.json"

python3 mlx-model-porting/scripts/scaffold_port.py \
  "$WORK/inspection.json" \
  --artifact-root "$MODEL" \
  --output "$WORK/mlx_port"
```

The version-locked `scaffold_port.py` 1.0.2 generator validates both config
semantics and inspected tensors. It accepts only `model_type="bert"` with a
supported BERT architecture identity, and rejects RoBERTa (whose padding-aware
position offset is not implemented), decoder/cross-attention flags,
relative/rotary positions, unknown computation keys even when falsey,
incomplete BERT layer inventories, and decoder, cross-attention, expert,
router, or unrecognized head tensors. The checked-in
[`scaffold-manifest.json`](scaffold-manifest.json) records the 199-parameter
target contract.

## Weight map and F32 conversion

```bash
python3 mlx-model-porting/scripts/convert_checkpoint.py \
  --source "$WORK/inspection.json" \
  --scaffold-manifest "$WORK/mlx_port/scaffold-manifest.json" \
  --emit-draft-map "$WORK/WEIGHT_MAP.draft.json"

python3 mlx-model-porting/scripts/validate_weight_map.py \
  --source "$WORK/inspection.json" \
  --target "$WORK/mlx_port/scaffold-manifest.json" \
  --mapping mlx-model-porting/examples/worked-port-bge-base-en/WEIGHT_MAP.json \
  --output "$WORK/weight-map-validation.json"

python3 mlx-model-porting/scripts/convert_checkpoint.py \
  --source "$MODEL" \
  --mapping mlx-model-porting/examples/worked-port-bge-base-en/WEIGHT_MAP.json \
  --output "$WORK/converted-f32"
```

[`WEIGHT_MAP.json`](WEIGHT_MAP.json) is schema 2, non-draft, fully resolved,
and declares F32 for every target. [`weight-map-validation.json`](weight-map-validation.json)
records 200/200 source tensors accounted for and 199/199 target tensors mapped.

## Masked source and target captures

```bash
python3 mlx-model-porting/scripts/capture_oracle.py "$MODEL" \
  --mode encoder \
  --token-ids 101 7592 2088 999 \
  --attention-mask 1 1 1 0 \
  --generate-steps 0 \
  --output "$WORK/source-oracle.npz" \
  --manifest "$WORK/source-oracle.manifest.json"

python3 mlx-model-porting/scripts/capture_mlx.py \
  --mode encoder \
  --package "$WORK/mlx_port" \
  --weights "$WORK/converted-f32" \
  --token-ids 101 7592 2088 999 \
  --attention-mask 1 1 1 0 \
  --generate-steps 0 \
  --output "$WORK/target-f32.npz" \
  --manifest "$WORK/target-f32.manifest.json"
```

Both capture paths preserve raw integer IDs and masks exactly. They zero padded
query rows only in retained floating sequence tensors; the model computation
itself preserves ordinary BERT key-mask semantics. `capture_mlx.py` validates
the conversion-report schema and enforces its weights and target-manifest
digests before loading the model. There is no generated-token rung,
autoregressive loop, or KV cache.

## Parity ladder

```bash
python3 mlx-model-porting/scripts/run_parity.py \
  --source-model "$MODEL" \
  --package "$WORK/mlx_port" \
  --weights "$WORK/converted-f32" \
  --mode encoder \
  --token-ids 101 7592 2088 999 \
  --attention-mask 1 1 1 0 \
  --generate-steps 0 \
  --atol 5e-5 --rtol 1e-4 \
  --output "$WORK/parity-report.json"
```

[`parity-report.json`](parity-report.json) contains the encoder contract mode,
token-ID input mode, exact attention mask, and every per-rung metric. The test
suite also runs the unequal-length prompts `hello` and `hello world` as a padded
`[2, 4]` batch, then uses the explicit target fault injector to prove the
first-divergence ladder stops at `layer.4.hidden` without accepting stale
conversion attestations.

## Checked-in artifacts and exclusions

- `inspection.json`: local static intake and pinned tensor inventory.
- `scaffold-manifest.json`: generated target parameter contract.
- `WEIGHT_MAP.json` and `weight-map-validation.json`: complete authored map and
  coverage proof.
- `source-oracle.manifest.json` and `target-f32.manifest.json`: capture metadata
  without tensor payloads.
- `converted-target-manifest.json`: converted F32 key/shape/dtype inventory.
- `parity-report.json`: passing real-model ladder.

No weights, NPZ tensors, generated package, or file larger than 1 MiB is
checked in. ViT, DINOv2, SigLIP, and CLIP vision frontends remain outside this
text-BERT worked port even though they share the broader encoder family route.
